1. 目的（Why）

上京後、日々の仕事や移動の中で忘れてしまいがちな故郷・桜島を、
「立ち止まる時間」だけでも身体的に思い出す体験を提供する。

忙しく歩き続ける状態＝忘却

立ち止まる状態＝想起

を、スマートフォンを通じて可視化する。

2. コンセプト（What）
基本構造

歩くと桜島が徐々に崩れ、最終的に霧散する

立ち止まると桜島が形成される

桜島は破壊されるのではなく、
意識の外へ「薄れていく」／「戻ってくる」存在として扱う。

3. 体験仕様（UX要件）
入力（ユーザー行為）

スマホを持って歩く

立ち止まり、数秒静止する

※視線・操作ボタン・タップ操作は使わない（極力排除）

出力（視覚変化）
歩行中

桜島の点群が徐々に透明化

ノイズにより霧のように拡散

輪郭が曖昧になる

停止中（一定時間後）

点群がゆっくり集まり始める

透明度が回復

山の輪郭が再形成される

4. 判定ロジック（機能要件）
歩行判定

加速度センサの変化量が一定以上

状態：isWalking = true

停止判定

加速度・向き変化が小さい状態が3秒以上継続

状態：isStill = true

※一瞬の停止では切り替わらない（ヒステリシス必須）

5. 表現仕様（ビジュアル要件）

表現手法：three.js + Points（点群）

点数：20,000〜50,000点（スマホ安定動作）

色：灰色〜白系の単色

透明度変化あり

霧感演出：
transparent = true

depthWrite = false

背景：黒または暗色

6. 技術要件（非機能要件）

対応端末：スマートフォン（iOS / Android）

実行環境：モバイルブラウザ

専用デバイス不要

7. スコープ外（やらないこと）

UIボタン・操作説明の多用

高精細テクスチャ

音・BGM（今回は不要）

科学的に正確な噴火表現
